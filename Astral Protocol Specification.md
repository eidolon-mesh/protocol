# **Astral Protocol Specification (Backend Routing & Control)**

**Version:** 1.0 (Eidolon Draft)

**Sub-Protocols:** Tether (JSON Control Plane) & Leyline (Binary Data Plane)

**Purpose:** Defines the server-to-server communication standards for Project Eidolon, handling node federation, global state synchronization, and cross-node packet routing.

## **1\. Protocol Overview**

The astral library manages all communication that does *not* involve a client (shard). All Astral connections mandate mutually authenticated TLS (mTLS) or Secure WebSockets (WSS) backed by strong API keys/Bearer Tokens.

* **Tether:** Nexus ↔ Aether (Slow, JSON-based, Control State).  
* **Leyline:** Aether ↔ Aether (Extremely Fast, Binary-based, Data Transit).

## **2\. The Tether Protocol (Control Plane)**

Tether is used by Aether nodes to report their health to Nexus, and by Nexus to distribute global routing tables and Magic DNS updates.

**Transport:** Secure WebSockets (wss://) using JSON payloads.

### **Authentication Flow**

When an Aether node boots, it opens a connection to Nexus:

{"type": "AUTH", "node\_id": "aether-us-east-1", "token": "sk\_live\_..."}

Nexus validates the token and responds with a subnet assignment.

### **Packet Types**

* ROUTE\_SYNC (Nexus \-\> Aether)  
  * Pushes the latest .mesh DNS records and Aether IP-to-Node mappings.  
  * *Payload:* List of active subnets and their hosting Aether URLs.  
* TELEMETRY (Aether \-\> Nexus)  
  * Sent every 10 seconds.  
  * *Payload:* {"active\_clients": 402, "cpu\_load": 0.45, "bandwidth\_mbps": 120}  
* ADMIN\_CMD (Nexus \-\> Aether)  
  * Executes Trust & Safety or maintenance commands.  
  * *Payload:* {"action": "kick\_user", "target\_ip": "fd00:3::abcd"} or {"action": "flush\_dns", "domain": "malicious.internal.eidolonmesh.org"}

## **3\. The Leyline Protocol (Data Plane)**

Leyline is used when Client A (connected to Aether-1) wants to send a packet to Client B (connected to Aether-2). It is designed for maximum throughput with zero JSON-parsing overhead.

**Transport:** Secure WebSockets (wss://) using binary frames.

### **Packet Structure**

\[TYPE (1 Byte)\] \[TARGET\_IPV6 (16 Bytes)\] \[SOURCE\_IPV6 (16 Bytes)\] \[PAYLOAD (...)\]

### **Packet Types**

#### **0x01 ENCAPSULATED\_WISP**

* **Purpose:** Blindly shovels Wisp frames (both standard 0x01 CONNECT and Vellum 0x11 DATAGRAM packets) from one Aether node to another.  
* **Payload:** The exact, unmodified Wisp frame generated by the client.  
* **Routing:** Receiving Aether node reads the TARGET\_IPV6, looks up the client's local WebSocket, and dumps the payload into it.

#### **0x02 WEBRTC\_OFFER\_PROXY**

* **Purpose:** Passes ICE candidates and Session Description Protocol (SDP) strings across the network so two clients can attempt a direct P2P puncture, eventually bypassing the servers entirely.  
* **Payload:** The raw SDP string.

#### **0x03 NODE\_STATUS**

* **Purpose:** Graceful degradation.  
* **Payload:** \[Status Code (1 Byte)\]  
* **Codes:** 0x01 (Shutting Down), 0x02 (Overloaded / Shedding Load).  
* **Action:** If Aether-2 sends 0x01 Shutting Down to Aether-1, Aether-1 knows to buffer packets or route new connections to a different node.

## **4\. Cross-Node Data Flow Example**

1. **Client A** (fd00::A) on **Aether-US** sends a Vellum 0x11 DATAGRAM to **Client B** (fd00::B).  
2. **Aether-US** checks its routing table (synced via Tether) and sees fd00::B is connected to **Aether-EU**.  
3. **Aether-US** wraps the 0x11 packet in a Leyline 0x01 ENCAPSULATED\_WISP frame.  
   * *Frame:* \[0x01\] \[fd00::B\] \[fd00::A\] \[Original 0x11 Packet\]  
4. **Aether-US** fires this frame down its persistent binary WebSocket to **Aether-EU**.  
5. **Aether-EU** unwraps the Leyline frame, sees the target is fd00::B, and drops the inner 0x11 packet down Client B's WebSocket.
